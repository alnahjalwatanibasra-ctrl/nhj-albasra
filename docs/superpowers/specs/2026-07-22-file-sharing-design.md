# تصميم ميزة «مشاركة الملفات» — Nhj AL-Basra

> **الحالة:** مُعتمَد من المستخدم عبر جلسة عصف ذهني (2026-07-22). جاهز لخطة التنفيذ.
> **الإصدار المقترح للنشر:** 4.0 (ميزة كبرى جديدة). المنشور حالياً 3.7.

---

## 1) الهدف والسياق

مكتب «تحالف النهج الوطني — البصرة» يستخدم التطبيق على **جهازين** (قابل للتوسّع لأكثر) على **نفس الشبكة المحلية**، والعمل مشترك بينهم مع تبادل مستمر للملفات. حالياً يستخدمون **واتساب** لتبادل الملفات، وهو غير عملي (ضغط الصور، عبر الهاتف، فوضى).

**الهدف:** إضافة ميزة داخل التطبيق لمشاركة الملفات مباشرةً بين الأجهزة على الشبكة المحلية، تكون بديلاً عملياً عن واتساب.

**غير الأهداف (خارج النطاق):**
- المشاركة عبر الإنترنت بين مواقع متفرقة (الأجهزة دائماً على نفس الشبكة).
- جهاز مركزي/خادم دائم التشغيل.
- تخزين سحابي أو أي تكلفة شهرية.
- رمز اقتران/تشفير معقّد (الشبكة خاصة بهم).

## 2) القرارات المعتمدة (من جلسة العصف الذهني)

| البند | القرار |
|------|--------|
| **طوبولوجيا الشبكة** | شبكة محلية واحدة (نفس المكتب)، شبكة خاصة بهم |
| **المعمارية** | **نِدّ لِنِدّ (P2P) بلا جهاز مركزي** — كل جهاز يرسل ويستقبل بالتساوي |
| **عدد الأجهزة** | جهازان حالياً، كلاهما مشتغل معاً أثناء الدوام. يُصمَّم لـ N≥2 |
| **التعارف** | **تلقائي بلا إعدادات** ما دام نفس الشبكة (بثّ UDP على الشبكة المحلية) |
| **اسم الجهاز** | حقل اختياري في إعدادات «عام»؛ الافتراضي = اسم حاسوب ويندوز. يظهر كـ«صاحب الملف» |
| **نموذج المشاركة** | مساحة تعرض بيانات الملفات؛ البايتات تُجلب من جهاز صاحبها عند الطلب (lazy pull) |
| **موقع نسخة المشاركة** | **الخيار (أ): الإشارة للمسار الأصلي فقط، صفر نسخ.** لو اختفى الأصل → زر «تحديد الموقع الجديد» |
| **حفظ الملفات المنزَّلة** | `%APPDATA%\Nhj AL-Basra\الملفات المستلمة\` (تُفتح من داخل التطبيق + زر «فتح موقع الملف») |
| **الترتيب** | الأحدث أولاً |
| **التبويبات** | تبويبان: «المشتركة من الآخرين» (الوارد) + «ملفاتي المشاركة» (الصادر مني، عرض إداري خاص) |
| **الحذف** | صاحب الملف فقط، من «ملفاتي المشاركة» → يُزال من المساحة عند الجميع. النسخ المُنزَّلة سابقاً تبقى على أجهزة مالكيها |
| **حالة عدم الاتصال** | ملفات الزميل المطفأ **تبقى ظاهرة** مع وسم «غير متاح الآن» |
| **تحسين مرافق** | إعادة هيكلة «الإعدادات المتقدمة»: بدل مربّع معطّل ظاهر → مخفيّة خلف زر «⚙ إظهار الإعدادات المتقدمة» |

## 3) المعمارية التقنية

كود جديد معزول في `core/sharing/` وواجهة جديدة في `ui/`، **دون المساس** بميزتَي سجل الصادر واستخراج النصوص.

### 3.1 المكوّنات (وحدات مستقلة، كل منها مسؤولية واحدة)

**`core/sharing/discovery.py` — التعارف (Discovery)**
- كل تطبيق يبثّ **حزمة إعلان (beacon)** عبر UDP broadcast على منفذ ثابت كل ~٣ ثوانٍ، تحوي: `device_id` فريد، اسم الجهاز، منفذ خادم HTTP، و**ترويسة سحرية (magic header)** تميّز تطبيق Nhj (فلا يتفاعل إلا مع تطبيق مثله).
- يستمع لإعلانات الأقران ويبني **جدول أقران متصلين** مع طابع «آخر ظهور»؛ يُعَدّ القرين «غير متصل» إذا انقطع إعلانه أكثر من ~١٠ ثوانٍ.
- بلا أي مكتبة خارجية (socket القياسية فقط) — مهم لتغليف PyInstaller.

**`core/sharing/registry.py` — سجل ملفاتي المشاركة**
- تخزين محلي دائم (JSON في مجلد بيانات التطبيق) لقائمة ما يشاركه **هذا الجهاز**: `{id, path, name, size, type, shared_at}`.
- الملف الأصلي يبقى في مكانه (الخيار أ) — لا نسخ.
- دوال: `add(path)`, `remove(id)`, `list()`, `resolve_missing(id, new_path)` (لزر تحديد الموقع الجديد).

**`core/sharing/server.py` — خادم الملفات المحلي**
- خادم HTTP خفيف (من مكتبة بايثون القياسية، في خيط منفصل) يقدّم:
  - `GET /index` → JSON بقائمة ملفات هذا الجهاز المشاركة (id, name, size, type, shared_at, owner_name, **available**) — حيث `available` يُحسب بفحص وجود الأصل (`os.path.exists`) فيعرف القرين أن الأصل مفقود قبل أن يحاول التنزيل.
  - `GET /file/<id>` → يبثّ بايتات الملف من مساره الأصلي (٤٠٤ إن كان مفقوداً).
  - `GET /ping` → للتحقق من الحياة.
- **أمان:** يقدّم فقط الملفات الموجودة في السجل، والوصول **بالمعرّف (id) لا بالمسار الخام** من العميل (منع اجتياز المسارات path traversal).

**`core/sharing/client.py` — عميل الجلب**
- يستعلم دورياً عن `GET /index` لكل قرين متصل ويبني قائمة «المشتركة من الآخرين».
- `download(peer, file_id)` → يجلب `/file/<id>` مع **تقدّم**، ويحفظ في مجلد «الملفات المستلمة».
- منطق تحديد الحالة (متاح/غير متاح/مُنزَّل).

### 3.2 الواجهة (`ui/`)
- **الصفحة الرئيسية:** بطاقة/زر جديد «مشاركة الملفات» بهوية النهج (تركوازي `#13908C`/`#19BBBD`، ليموني `#E6EC26`).
- **`ui/sharing_page.py`:** صفحة المساحة، وفيها:
  - تبويبان أعلى: «المشتركة من الآخرين» و«ملفاتي المشاركة».
  - منطقة بيضاء للسحب والإفلات + زر «اختيار ملف».
  - صفوف الملفات: أيقونة نوع + اسم + صاحب + تاريخ + حجم + حالة + أزرار إجراء (تنزيل/فتح/إزالة/تحديد الموقع).
  - مؤشّر «أجهزة متصلة» + رسالة عند عدم وجود أقران.
- **إشعار خفيف** عند وصول ملف جديد (شارة على زر المشاركة).

### 3.3 الإعدادات (`ui/dialogs/settings_dialog.py`)
- **إضافة حقل «اسمك»** في قسم «عام» (يظهر لزملائك عند مشاركة ملف). الافتراضي = اسم الحاسوب.
- **إعادة هيكلة «متقدم»:** إزالة نمط `QGroupBox` القابل للتفعيل الذي يظهر معطّلاً؛ استبداله بعناصر **مخفيّة** تُكشَف بزر/سطر في النهاية «⚙ إظهار الإعدادات المتقدمة».

### 3.4 التخزين والإعدادات (`core/config.py`)
- مفاتيح جديدة: `device_name`, وإعدادات المشاركة (المنفذ إن لزم). مجلد «الملفات المستلمة» و`registry.json` ضمن `%APPDATA%\Nhj AL-Basra\`.

## 4) تدفّق البيانات

**المشاركة (فوري):**
1. المستخدم يسحب ملفاً → `registry.add(path)` → يظهر فوراً في «ملفاتي المشاركة».
2. القرين يستعلم `GET /index` دورياً → يظهر الملف في «المشتركة من الآخرين» عنده خلال ثوانٍ.

**التنزيل (عند الطلب):**
1. الزميل يضغط ملفاً في «المشتركة من الآخرين» → `GET /file/<id>` مع شريط تقدّم.
2. يُحفظ في «الملفات المستلمة» → الحالة تصبح «مُنزَّل» + زر «فتح».

**الإزالة:**
1. صاحب الملف يزيله من «ملفاتي المشاركة» → `registry.remove(id)`.
2. عند استعلام القرين التالي لا يعود الملف في `/index` → يختفي من مساحته. النسخة المنزَّلة سابقاً تبقى على قرصه.

**تحديد التوفّر:**
- القرين متصل (إعلانه حديث) + الملف مدرَج في `/index` → «متاح» 🟢.
- القرين غير متصل → «غير متاح الآن» ⚪ (يبقى ظاهراً).
- القرين متصل لكن الأصل مفقود → صاحبه يرى «الملف لم يعد موجوداً» + زر «تحديد الموقع الجديد»؛ القرين يرى «غير متاح».

## 5) معالجة الأخطاء والحالات الحدّية

| الحالة | السلوك |
|--------|--------|
| القرين يُطفئ جهازه أثناء التنزيل | «انقطع الاتصال، حاول لاحقاً»؛ الملف الجزئي يُحذف؛ الحالة «غير متاح الآن» |
| لا يوجد أي قرين على الشبكة | «لا يوجد أجهزة متصلة حالياً» |
| الأصل نُقل/حُذف/غُيّر اسمه (الخيار أ) | صاحبه: «الملف لم يعد موجوداً عند صاحبه» + زر «تحديد الموقع الجديد» (اختيار واحد) |
| جدار حماية ويندوز يمنع الاتصال | أول تشغيل: سؤال ويندوز مرة واحدة «سماح»؛ إن مُنع → رسالة إرشادية عن السماح على الشبكة |
| المنفذ مشغول | محاولة منافذ بديلة تلقائياً |

## 6) الاختبار

- **وحدات (pytest، ضمن `tests/`):** إضافة/إزالة/بقاء السجل؛ تسلسل `/index`؛ تحليل حزمة الإعلان؛ منطق التوفّر؛ ربط النوع بالأيقونة؛ الترتيب حسب التاريخ؛ منع اجتياز المسارات.
- **تكامل:** خادمان داخل نفس العملية → دورة مشاركة←index←تنزيل؛ انتشار الإزالة؛ معالجة الملف المفقود.
- يتّبع بنية الاختبارات الحالية (٥٧ اختباراً).

## 7) الأمان والخصوصية

- شبكة محلية خاصة؛ **ترويسة سحرية** فلا يتفاعل إلا تطبيق Nhj مع مثله.
- تُقدَّم فقط ملفات السجل، بالمعرّف لا بالمسار الخام (منع اجتياز المسارات).
- لا انكشاف على الإنترنت (ربط على الشبكة المحلية فقط).
- بيانات المكتب حسّاسة؛ لا يخرج شيء خارج الشبكة المحلية.

## 9) تحسينات مؤجَّلة (YAGNI — ليست في v1)

- تخزين مؤقت ذكي (caching) ليبقى الملف متاحاً حتى لو أُطفئ جهاز صاحبه.
- إدخال يدوي لعنوان الجهاز (fallback) لو فشل التعارف التلقائي على شبكة غريبة.
- المشاركة عبر الإنترنت / جهاز مركزي.
