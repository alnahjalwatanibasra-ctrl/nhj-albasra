# وثيقة تصميم: الواجهة الرسومية لبرنامج «سجل الصادر» (المرحلة ٢)

> **الحالة:** معتمدة من المستخدم بعد نقاش تفصيلي ونماذج بصرية (2026-07-14).
> **الأساس:** المحرّك (المرحلة ١) جاهز ومختبَر بدقة 100% على مجموعة الاختبار — هذه الوثيقة تغطي الواجهة فقط.

## 1) الهدف والمستخدمون

تطبيق ويندوز مستقل (`.exe` لاحقاً) يغلّف محرّك الاستخراج بواجهة عربية RTL بهوية «تحالف النهج الوطني».
- **المستخدمون:** عدة موظفين، بعضهم غير تقني. **الاستخدام متباعد** (كل أسبوع–أسبوعين أو أكثر) — الواجهة يجب أن تذكّر الموظف الناسي بتسلسل العمل بنفسها (خطوات مرقّمة ظاهرة).
- **الوضعان:** مبسّط افتراضياً + قسم «متقدم» مطوي في الإعدادات.

## 2) قرارات المستخدم الملزِمة

| البند | القرار |
|---|---|
| بنية الواجهة | **شاشة واحدة متحوّلة** بخطوات مرقّمة ①②③ (لا معالج متعدد الشاشات ولا تبويبات) |
| التسمية | «انجازات النهج» يُسمى في الواجهة **«الملف المرجعي»** |
| المسارات (مرجعي/سجل سابق/مجلد Word) | محفوظة من آخر مرة، يعرض كلٌّ منها المسار الحالي + زر **«تغيير»** |
| صورة السجل أثناء المراجعة | ليست مقسومة دائماً — زر **«عرض الصورة»** يفتحها عند الحاجة |
| رقم الكتاب الأول | من **ملف السجل السابق** الذي يختاره الموظف (يكمل من آخر رقم فيه) |
| هواتف Word | **نافذة مستقلة** بعد الاستخراج تعرض كل الاقتراحات للتأكيد سطراً سطراً |
| التصدير | **ملف Excel جديد مستقل لكل عملية** (لا إلحاق بسجل رئيسي) |
| مقترحات معتمدة | حفظ الجلسة تلقائياً ✚ تمييز الصفوف غير المطابقة للمرجع ✚ زر فتح مجلد الملف بعد التصدير |
| مقترحات مرفوضة | عدّاد الحصة اليومية ✗ • تحذير جودة الصورة ✗ |

## 4) الشاشات (نماذجها عُرضت واعتُمدت)

### 4.1 شاشة البداية
- شريط علوي `#13908C`: شعار مصغّر + «سجل الصادر — تحالف النهج الوطني» + زر ⚙.
- **① صور السجل:** بطاقة سحب/إفلات (وأيضاً نقر لفتح منتقي ملفات؛ JPG/PNG متعددة). الصور المضافة تظهر **مصغّرات مرقّمة** بترتيبها (يؤثر على الترقيم التسلسلي) مع زر ✗ للحذف وإمكانية إعادة الترتيب بالسحب. تلميح رمادي: «الصور الأوضح تعطي نتائج أدق — أرسلها من واتساب كملف لا كصورة».
- **② الملفات:** ثلاثة أسطر (الملف المرجعي، السجل السابق، مجلد طلبات Word «اختياري») بمساراتها المحفوظة + زر «تغيير». مسار محفوظ لم يعد موجوداً ⟵ يظهر بالأحمر «⚠ الملف غير موجود». ترك مجلد Word فارغاً = تخطّي ميزة اقتراح الهواتف.
- **③ زر «▶ ابدأ الاستخراج»** ليموني كبير. معطّل حتى: صورة واحدة على الأقل + ملف مرجعي صالح، مع نص يوضح سبب التعطيل.
- **السجل السابق اختياري عملياً:** إن حُدِّد يكمل الترقيم من آخر رقم فيه (قرار المستخدم)؛ إن تُرك فارغاً يستنتج المحرّك رقم البداية من أرقام الصور تلقائياً (السلوك الاحتياطي المُختبر).

### 4.2 أثناء الاستخراج
- شريط تقدم تركوازي + حالة نصية («جاري استخراج الصورة ٢ من ٣...») + المراحل التالية مذكورة + زر «إيقاف».
- أخطاء بالعربية الواضحة: انقطاع إنترنت، نفاد حصة («جرى التحويل للنموذج الاحتياطي»)، فشل نهائي بعد كل المحاولات. المحرّك يعيد المحاولة ويتناوب النماذج تلقائياً (سلوك موجود).

### 4.3 شاشة المراجعة
- تتصدرها **عدّادات**: عدد الصفوف، المؤكد من المرجع، و(المقترح المعتمد ٤) شريط تحذير إن وُجدت صفوف غير مطابقة: «⚠ N صفوف لم تُعثر في الملف المرجعي» + زر «التالي» يقفز بينها.
- **جدول** بأعمدة المخرجات الثمانية، خلاياه ملوّنة بألوان الثقة، **تحرير مباشر بالنقر المزدوج**. الخلية المعدّلة يدوياً يزول لونها (قرار بشري).
- شريط أدوات: «تأكيد الهواتف (N)» يفتح نافذة 4.4 • «عرض الصورة» يفتح صورة الصفحة المقابلة للصف المحدد في نافذة جانبية • وفي أقصى الطرف الآخر منفصلاً: **«تصدير Excel»** ليموني (مفصول عمداً كي لا يُضغط سهواً قبل إتمام المراجعة).
- مفتاح ألوان صغير أسفل الجدول (4 ألوان الثقة).

### 4.4 نافذة تأكيد هواتف Word (حوارية مستقلة)
لكل اقتراح: الاسم • الهاتف المقترح • **اسم ملف المصدر** • أزرار: «✓ اعتماد» / «فتح الملف» (يفتح docx بالوورد للتحقق) / «✗ رفض».
قاعدة موروثة ملزمة: لا اعتماد تلقائي أبداً — التأكيد بشري لكل سطر.

### 4.5 التصدير
- حوار حفظ باسم مقترح تلقائياً: `سجل_الصادر_<أول رقم>-<آخر رقم>.xlsx`، والتصدير بمحرّك `excel_export` الموجود (بالألوان).
- بعد النجاح: رسالة «تم التصدير» + زرّا **«فتح الملف»** و**«فتح المجلد»** (المقترح المعتمد ٥).
- إن وُجدت اقتراحات هواتف لم تُراجع: تنبيه لطيف قبل التصدير («توجد N اقتراحات هواتف لم تُؤكد — تصدير على أي حال؟»).
- ملف مقفول بالـ Excel (PermissionError) ⟵ رسالة «أغلق الملف في Excel ثم أعد المحاولة».

### 4.6 نافذة الإعدادات (⚙)
- **عام:** مفتاح Gemini (معروض مقنّعاً `AQ.Ab8••••`، زر تغيير) + زر «فحص الاتصال والحصة الآن».
- **متقدم (مطوي افتراضياً، بعنوان «لا تغيّره إلا إذا كنت تعرف ما تفعل»):**
  - **استبدالات المصطلحات:** نافذة جدول ثنائي (ما يُقرأ خطأً ← الصواب) بإضافة/حذف صفوف — تُطبق تلقائياً على «الموضوع» (عملياً تنفع الصفوف غير المطابقة للمرجع؛ المطابَقة تأخذ موضوعها من المرجع).
  - النماذج وترتيبها (قائمة قابلة للتحرير).
  - مفتاح تشغيل/إيقاف «إرشاد القراءة بمفردات المرجع» (`vocab_in_prompt`).
  - عتبات المطابقة (أرقام، للخبراء فقط).
- كل الإعدادات في `settings.json` (آلية `core/config.py` الموجودة).

## 5) المقترح المعتمد ٢: حفظ الجلسة تلقائياً

- بعد الاستخراج وبعد **كل تعديل** في الجدول: حفظ صامت لملف جلسة (`session_autosave.json` في مجلد البرنامج) يتضمن: نتائج الاستخراج، التعديلات اليدوية، حالة تأكيدات الهواتف، مسارات الصور.
- عند فتح البرنامج ووجود جلسة غير مصدَّرة: «لديك جلسة لم تُصدَّر (N صفاً) — استكمالها؟» نعم يفتح شاشة المراجعة مباشرة (بلا أي استهلاك للحصة) / لا يحذفها بعد تأكيد.
- التصدير الناجح يمسح ملف الجلسة.

## 6) البنية التقنية

- **PySide6** (Qt) — نافذة رئيسية بـ `QStackedWidget` بثلاث حالات (بداية/تقدم/مراجعة)، `QTableWidget` للجدول، حوارات `QDialog` للهواتف والإعدادات والاستبدالات.
- **RTL:** `setLayoutDirection(Qt.RightToLeft)` على مستوى التطبيق + `QLocale` عربية.
- **الثيم:** ملف QSS واحد (`ui/theme.qss`) يحمل ألوان الهوية أعلاه حصراً.
- **الخيط:** الاستخراج في `QThread` منفصل (المحرّك مطوّل النداءات) يبثّ إشارات progress/finished/error — الواجهة لا تتجمد أبداً، وزر «إيقاف» يرفع علم إلغاء يفحصه المحرّك بين الصور.
- **الفصل:** الواجهة تستدعي `core/pipeline.run` كما هو **دون أي تعديل على المحرّك** (الواجهة مستهلك فقط). أي نقص يظهر يُحل بإضافة معاملات اختيارية للمحرّك لا بمنطق داخل الواجهة.
- **هيكل الملفات:**
  ```
  ui/
    __init__.py
    app.py            (نقطة التشغيل QApplication + الثيم + RTL)
    main_window.py    (النافذة والحالات الثلاث)
    start_page.py     (① ② ③)
    progress_page.py
    review_page.py    (الجدول + الأدوات + عرض الصورة)
    dialogs/
      phones_dialog.py
      settings_dialog.py
      replacements_dialog.py
    session.py        (حفظ/استرجاع الجلسة)
    theme.qss
  assets/logo.jpg
  run_app.py          (تشغيل: python run_app.py)
  ```

## 7) معالجة الأخطاء (كلها برسائل عربية مفهومة)

| الحالة | السلوك |
|---|---|
| لا إنترنت | رسالة + زر إعادة محاولة |
| نفاد حصة النموذج | يتحول تلقائياً للاحتياطي ويُعلم الموظف بشريط معلومات (غير معطِّل) |
| فشل كل النماذج | رسالة «تعذر الاستخراج الآن — أعد المحاولة لاحقاً» مع بقاء الصور محمّلة |
| ملف مرجعي تالف/بلا أعمدة معروفة | رسالة تحدد المشكلة («لم يُعثر على عمود الاسم») |
| ملف التصدير مفتوح في Excel | «أغلق الملف ثم أعد المحاولة» |
| إغلاق البرنامج أثناء مراجعة غير مصدَّرة | تأكيد + الجلسة محفوظة تلقائياً على أي حال |

## 8) الاختبار

- **يدوي موجّه:** سيناريو كامل بصور الاختبار الثلاث مقابل الملف الذهبي (النتيجة يجب أن تبقى 100%).
- **آلي:** اختبارات وحدات لـ `session.py` (حفظ/استرجاع/مسح) ولربط الواجهة بالمحرّك عبر كائن نتائج جاهز (بلا نداءات Gemini — بالكاش).
- **حالات حافة:** جلسة محفوظة تالفة، صور تُحذف من القرص بين الحفظ والاسترجاع، مسارات ملفات بأحرف عربية ومسافات.

## 9) خارج نطاق هذه المرحلة

- التغليف `.exe` بـ PyInstaller (المرحلة ٣ — بعد اعتماد الواجهة).
- عدّاد الحصة وتحذير جودة الصورة (رفضهما المستخدم).
- أي تعديل على خوارزميات المحرّك.
