# وثيقة تصميم: ميزة «استخراج النصوص» + الشاشة الرئيسية (الإصدار 1.3)

> **الحالة:** معتمدة من المستخدم (2026-07-17) بعد نقاش سؤالاً بسؤال.

## 1) الميزة

قارئ عام: صور أوراق ⟵ النص الخام كما هو. **بلا** مرجع، بلا ألوان، بلا إعدادات خاصة، بلا جلسات.

| البند | القرار |
|---|---|
| الاسم | **«استخراج النصوص»** |
| الإدخال | عدة صور دفعة واحدة (سحب/اختيار) — تُستخرج **بالتوازي** |
| شكل النص | أسطر الورقة كما هي؛ الجداول ⟵ سطر لكل صف بفواصل « \| » بين الخلايا |
| تعدد الصور | نصوص متتالية في مربع واحد بفاصل «——— الصفحة N ———» |
| الأفعال | زر **«نسخ الكل»** + تحديد حر بالمؤشر للنسخ الجزئي |
| البنية التحتية | نفس محرّك Gemini (تصغير الصور، تناوب النماذج، مهلة 300ث) بتعليمات نصية مبسطة |

## 2) الشاشة الرئيسية (المدخل الجديد)

- البرنامج يفتح **دائماً على الشاشة الرئيسية**: بطاقتان كبيرتان **بأسمائهما فقط بلا شرح**:
  «سجل الصادر» و«استخراج النصوص». تتسع لبطاقات مستقبلية.
- استثناء وحيد: جلسة سجل صادر غير مصدَّرة ⟵ سؤال «استكمالها؟» ينقل للمراجعة مباشرة.
- زر **«⌂ الرئيسية»** في الشريط العلوي داخل أي ميزة (مخفي في الرئيسية نفسها)،
  مع تنبيه عند مغادرة مراجعة غير مصدَّرة (تبقى محفوظة تلقائياً).
- الإعدادات (⚙) مشتركة واحدة للميزتين.
- سجل الصادر لا يتغير فيه شيء — فقط يُدخل إليه من بطاقته.

## 3) شاشة «استخراج النصوص»

منطقة سحب صور (نفس مكوّن سجل الصادر) ⟵ زر ليموني «استخراج النصوص» ⟵ حالة تقدم
مدمجة بعدّاد وقت ⟵ مربع نص كبير قابل للتحديد + «نسخ الكل». زر «صور جديدة» يفرغ الشاشة.

## 4) تقني

- `core/gemini_ocr.py`: دالة `extract_text_image` بـ prompt نصي (بلا JSON) تشارك
  التصغير والتناوب وذاكرة النماذج الممتلئة.
- `ui/home_page.py` بطاقات، `ui/text_page.py` شاشة الميزة (حالتها الداخلية كاملة —
  لا تشارك صفحة تقدم سجل الصادر)، `ui/text_worker.py` خيط بتوازي الصور.
- `main_window`: إضافة الصفحتين للمكدس + زر الرئيسية + الإصدار **1.3**.
- خارج النطاق: حفظ كملف، جلسات لهذه الميزة، أي معالجة لاحقة للنص.
