## 1) ما هو المشروع

**ميزتان:**

- **استخراج النصوص:** صور/مسح ضوئي ← نص خام قابل للنسخ. يشمل **مسحاً ضوئياً مباشراً** من داخل البرنامج (Canon MF750C عبر WIA).

## 3) البنية التقنية
- **المحرّك** `core/`: `pipeline` (المحاذاة التسلسلية + سحب كل حقول المرجع + الدائرة من المعرّف)، `reference`, `gemini_ocr` (بلا «تفكير» thinking=0 = أسرع 15×)، `corrections`, `excel_export`, `word_phones`, `scanner` (WIA), `updater`, `version`, `config`.
- **الواجهة** `ui/`: PySide6 RTL. `home_page` (بطاقتان)، `start_page`+`review_page` (سجل الصادر)، `text_page`+`text_worker`+`scan_worker` (النصوص+المسح)، `dialogs/`، `update_flow`, `main_window`.
- **الهوية:** تركوازي `#13908C`/`#19BBBD`، ليموني `#E6EC26`، خط «نور» و«المخطوطة» و«الشعار» في `assets/`. المفتاح مضمّن في `assets/keys/` (مُتجاهَل من git).
- **الاختبارات:** `python -m pytest tests/` (57 اختباراً). المحرّك: `python run_engine.py` (يقارن بالذهبي، يجب 100%).

## 4) نظام التحديث التلقائي (مهم جداً — فهمه ضروري)
- **الرابط الثابت:** `https://github.com/alnahjalwatanibasra-ctrl/nhj-albasra/releases/latest/download/NhjALBasra.exe` (و`version.json`). مدمج افتراضياً في `core/version.py` (`MANIFEST_URL`) — لا يعتمد على إعداد محفوظ.
- **النشر التلقائي:** `git tag vX.Y && git push origin vX.Y` ← **GitHub Actions يبني وينشر تلقائياً** (`.github/workflows/release.yml`). سرّ `GEMINI_KEY` مضبوط في إعدادات المستودع.
- **دورة الإصدار:** ارفع الرقم في `core/version.py` ← `RELEASE_NOTES.txt` ← commit ← push main ← `git tag vX.Y` ← `git push origin vX.Y` ← انتظر Actions ← تأكد بجلب المانيفست.
- **⚠️ قاعدة ذهبية:** كود **إعادة التشغيل** (`apply_and_restart`) يعمل في النسخة التي **تُجري** التحديث (القديمة). فأي إصلاح لإعادة التشغيل لا يظهر أثره إلا في التحديث **التالي** بعد وصول النسخة التي تحمله.

## 5) رحلة إصلاح التحديث (ما جُرّب — لا تكرّره)
تسلسل الأعطال المكتشفة وحلولها (كلها في 3.7 الآن):
1. **رابط فارغ محفوظ** يعطّل التحديث ← `config.manifest_url()` يستخدم المدمج دائماً.
2. **نافذة التنزيل لا تُغلق** (خطأ `x or dlg.close()` — الاختصار يمنع الإغلاق) ← دوال صريحة.
3. **تنزيل ناقص يُثبَّت** ← فحص صارم: الحجم = المعلن + رأس `MZ`.
4. **تعليق التنزيل** ← مهلة مقبس + رسالة خطأ.
5. **خطأ «Failed to load Python DLL» بعد إعادة التشغيل** ← **السبب الجذري:** تسريب متغيّر `_MEIPASS2` للـ exe الجديد. الحل: تنظيف بيئة `_MEI*`/`_PYI*` قبل التشغيل في البات + `env` نظيف (3.5). *الفتح اليدوي يعمل دائماً لأنه بلا هذا المتغيّر.*
6. **الفحص اليدوي يفشل دائماً** ← `w.wait()` يجمّد استقبال إشارة الخيط ← جعله متزامناً (3.7).
7. **لا فحص أثناء التشغيل** ← فحص دوري كل 30 دقيقة + كسر تخزين المانيفست (3.3).
